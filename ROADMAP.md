# Roadmap
Проект: ioBroker адаптер для Wiren Board RS485
Репозиторий: https://github.com/mihailkol/ioBroker.wirenboard
Текущее состояние:
Адаптер работает, поддерживает WB-M1W2, WB-MAI6, WB-MR3LV, WB-MAP3E. Есть таб с конфигурацией, сканированием шины, управлением реле. Основная проблема — задержка записи реле из-за конкуренции с поллингом за TCP соединение.

Архитектура транспорта:
Один TCP клиент на шлюз, приоритетная очередь запросов:
ПриоритетЧтоИнтервалP0Команды записи (coil, holding)немедленно, вытесняет поллингP1Дискретные входы, кнопки100-200мсP2Аналог, счётчики5-10сP3Системные данные (серийник, FW)при стартеP4Holding настройкипо запросу
При поступлении P0 — текущий запрос завершается, следующий не начинается пока команда не выполнена.

Структура шаблона — пересмотр:
Текущие шаблоны это адаптированный формат WB-MQTT-serial — сложный, нечитаемый, с jinja-макросами. Нужен собственный формат который:

Читается человеком без знания WB-формата
Явно описывает приоритет опроса каждого измерения
Разделяет measurements (ro), commands (rw coil), settings (rw holding)
Не требует jinja — чистый JSON
Содержит понятные id и имена на русском/английском
Описывает условия активности канала понятно ("condition": "in1_mode == 0")
